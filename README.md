# Uni-Mate Scaffold

Uni-Mate 입시 전략 서비스의 초기 구현 스캐폴드입니다. 제공된 HTML 화면을 기준으로 화면 흐름을 재정리하고, 프론트엔드와 백엔드/AI 로직이 나중에 쉽게 연결되도록 계약 중심 구조로 나눴습니다.

## 구성

- `frontend/`: Next.js App Router 기준 화면 스캐폴드
- `backend/`: FastAPI 연결 전제의 도메인/서비스 로직
- `ai/`: 모집요강 파싱, 근거 처리, 합격 가능성 스코어 로직
- `tests/`: 현재 환경에서 바로 실행 가능한 Python 단위 테스트

## 현재 포함된 화면

- 랜딩, 로그인
- 기본정보 입력, 성적입력, 성적표 업로드, 목표설정, 분석 진행중
- 메인 대시보드, 저장하기, 정보 없음, 가입 유도
- 전략추천, 추천 수강과목
- 전형분석, 갭분석, 시뮬레이션
- 실행관리, 설정
- 근거 보기, 회원가입

## 수정한 핵심 흐름

1. 랜딩의 `가입 없이 시작하기`는 바로 `기본정보 입력`으로 이동
2. 온보딩은 `기본정보 -> 성적입력 -> 목표설정` 순서를 강제
3. 성적입력 화면에서 PDF/OCR 업로드 진입점을 기본 제공
4. 분석 완료 후 메인 대시보드 진입
5. 대시보드에서 `전략`, `분석`, `실행`, `설정` 탭으로 이동
6. 모든 추천 결과에는 `근거 보기`가 연결되며 페이지 번호가 없는 경우 `확인 불가` 처리

## 테스트

```bash
python -m unittest discover -s tests -v
```

## 수동 QA

- `http://127.0.0.1:3001/qa`에서 목표대학 시나리오별 화면을 빠르게 점검할 수 있습니다.
